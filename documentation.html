Project Overview

Habitly is a premium habit tracking application built with Kotlin Multiplatform and Compose Multiplatform, designed to run seamlessly on both Android and iOS platforms. This sophisticated app helps users build and maintain positive habits with advanced analytics, beautiful UI, and comprehensive tracking features.

Target Platforms: Android 7.0+ (API 24+) and iOS 14.0+

Clean Architecture with MVVM pattern, dependency injection, and reactive programming

Premium UI/UX

Architecture Overview

Clean Architecture Implementation

Habitly follows Clean Architecture principles with clear separation of concerns:

Copy
📁 Project Structure:
├── 📂 core/
│   ├── 📂 data/           # Data layer (repositories, mappers)
│   ├── 📂 domain/         # Business logic (use cases, models)
│   ├── 📂 database/       # SQLDelight database
│   ├── 📂 prefs/          # DataStore preferences
│   ├── 📂 ui/            # UI components and theme
│   └── 📂 navigation/    # Navigation system
├── 📂 features/          # Feature modules
│   ├── 📂 home/          # Home screen
│   ├── 📂 statistics/    # Analytics screen
│   ├── 📂 settings/      # Settings screen
│   ├── 📂 addHabit/      # Habit creation
│   └── 📂 premium/       # Premium features
└── 📂 di/               # Dependency injection

Key Architectural Patterns

  • MVVM (Model-View-ViewModel): Reactive UI with state management
  • Repository Pattern: Abstract data access layer
  • Use Cases: Encapsulated business logic
  • Dependency Injection: Manual DI with singleton pattern
  • Flow-based Architecture: Reactive data streams

Database Schema

Table Purpose Key Fields
EntityHabit Habit definitions id, name, category, color, icon, targetCount, difficulty
EntityHabitDone Completion history id, habitId, completedDate, completedAt

Performance Optimizations

  • Optimized Repository: Single query with grouped data
  • Lazy Loading: Efficient list rendering
  • State Caching: Minimal recomposition
  • Background Processing: Coroutines for heavy operations
  • Memory Management: Proper lifecycle handling

Installation & Setup

Prerequisites

Requirements:
  • Android Studio Hedgehog (2023.1.1) or newer
  • JDK 17 or newer
  • Kotlin 2.2.10
  • Android SDK 34
  • Xcode 15+ (for iOS development)

Quick Start

Copy
# Clone the repository
git clone [your-repository-url]
cd habitly

# Open in Android Studio
# File → Open → Select habitly folder

# Sync Gradle files
# Click "Sync Now" when prompted

# Run the app
# Select "composeApp" run configuration
# Choose Android or iOS simulator

Project Setup Steps

  1. Import Project: Open Android Studio → Import Project
  2. Configure SDK: Set Android SDK path in local.properties
  3. Sync Dependencies: Let Gradle download all dependencies
  4. Configure Signing: Add your keystore for release builds
  5. Test Build: Run debug build on emulator/device

Troubleshooting Guide

Common Build Issues

Gradle Sync Problems

Issue: "Failed to sync Gradle project"
Copy
Solutions:
1. Clean and rebuild project:
./gradlew clean
./gradlew build

2. Invalidate caches:
File → Invalidate Caches and Restart

3. Check Gradle wrapper version:
Update gradle/wrapper/gradle-wrapper.properties

4. Verify JDK version (requires JDK 17+)

Performance Issues

Copy
Performance Optimization Checklist:
□ Use LazyColumn for large lists
□ Implement proper remember() for expensive calculations
□ Avoid unnecessary recompositions
□ Use derivedStateOf for computed states
□ Optimize database queries
□ Implement proper loading states
